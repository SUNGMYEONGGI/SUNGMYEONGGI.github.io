# DataLoader
> CLASS torch.utils.data.DataLoader(dataset, batch_size=1, shuffle=None, sampler=None, batch_sampler=None, num_workers=0, collate_fn=None, pin_memory=False, drop_last=False, timeout=0, worker_init_fn=None, multiprocessing_context=None, generator=None, *, prefetch_factor=None, persistent_workers=False, pin_memory_device='')

`DataLoader`는 데이터셋과 샘플러를 결합하여 주어진 데이터셋에 대한 반복 가능한 객체를 제공합니다.

`DataLoader`는 Map-Style과 Iterable-Style의 데이터셋을 모두 지원하며, 단일 프로세스와 다중 프로세스 로딩, 로딩 순서 커스터마이징, 콜레이션 및 메모리 핀닝을 지원합니다.

## Parameters
- batch_size (int, optional): 한 번에 로드할 배치의 샘플 수를 지정합니다. 기본값은 1입니다.

- shuffle (bool, optional): True로 설정하면 각 에포크마다 데이터가 섞입니다. 기본값은 False입니다.

- sampler (Sampler or Iterable, optional): 데이터셋에서 샘플을 추출하는 전략을 정의합니다. len 메서드가 구현된 Iterable이면 됩니다. 이 옵션을 지정하면 shuffle을 지정할 수 없습니다.

- batch_sampler (Sampler or Iterable, optional): sampler와 유사하지만, 한 번에 인덱스 배치를 반환합니다. batch_size, shuffle, sampler, drop_last와 상호 배타적입니다.

- num_workers (int, optional): 데이터 로딩에 사용할 서브프로세스 수를 지정합니다. 0으로 설정하면 메인 프로세스에서 데이터를 로드합니다. 기본값은 0입니다.

- collate_fn (Callable, optional): 샘플 리스트를 병합하여 미니 배치를 만듭니다. 맵 스타일 데이터셋에서 배치 로딩을 사용할 때 사용됩니다.

- pin_memory (bool, optional): True로 설정하면, 데이터 로더가 반환하기 전에 텐서를 CUDA 고정 메모리로 복사합니다. 데이터 요소가 사용자 정의 타입이거나 collate_fn이 사용자 정의 타입의 배치를 반환하는 경우 참고하십시오.

- drop_last (bool, optional): True로 설정하면, 데이터셋 크기가 배치 크기로 나누어떨어지지 않을 경우 마지막 불완전한 배치를 버립니다. False로 설정하면 마지막 배치가 더 작아집니다. 기본값은 False입니다.

- timeout (numeric, optional): 워커에서 배치를 수집하는 타임아웃 값을 지정합니다. 항상 0 이상이어야 합니다. 기본값은 0입니다.

- worker_init_fn (Callable, optional): None이 아니면, 각 워커 서브프로세스에서 데이터 로딩 전에 worker id를 입력으로 받아 호출됩니다. 기본값은 None입니다.

- multiprocessing_context (str or multiprocessing.context.BaseContext, optional): None이면, 운영체제의 기본 멀티프로세싱 컨텍스트가 사용됩니다. 기본값은 None입니다.

- generator (torch.Generator, optional): None이 아니면, RandomSampler가 무작위 인덱스를 생성하는 데 이 RNG가 사용되고 멀티프로세싱이 worker의 base_seed를 생성하는 데 사용됩니다. 기본값은 None입니다.

- prefetch_factor (int, optional, keyword-only arg): 각 워커가 미리 로드할 배치 수를 지정합니다. 2로 설정하면 총 2 * num_workers 배치가 모든 워커에서 미리 로드됩니다. num_workers 값에 따라 기본값이 다릅니다. num_workers가 0이면 기본값은 None입니다. 그렇지 않으면 기본값은 2입니다.

- persistent_workers (bool, optional): True로 설정하면, 데이터셋이 한 번 소비된 후에도 워커 프로세스를 종료하지 않습니다. 이는 워커의 Dataset 인스턴스를 유지시킵니다. 기본값은 False입니다.

- pin_memory_device (str, optional): pin_memory가 True인 경우 고정 메모리를 사용할 장치를 지정합니다.
