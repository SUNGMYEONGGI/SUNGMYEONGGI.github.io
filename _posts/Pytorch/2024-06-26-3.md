# torch.optim
`torch.optim`은 다양한 최적화 알고리즘을 구현하는 패키지입니다.

가장 일반적으로 사용되는 방법들이 이미 지원되며, 인터페이스는 충분히 일반적이어서 더 정교한 방법들도 쉽게 통합될 수 있습니다.

## How to use an optimizer
`torch.optim`을 사용하려면 현재 상태를 유지하고 계산된 그래디언트(기울기)에 기반하여 파라미터를 업데이트할 옵티마이저 객체를 생성해야 합니다.

### Constructing it
옵티마이저를 생성하려면 최적화할 파라미터들을 포함하는 iterable 객체를 제공해야 합니다(모두 Variable이어야 합니다). 그런 다음, 학습률(learning rate), 가중치 감쇠(weight decay) 등 옵티마이저 특화 옵션을 지정할 수 있습니다.
```python
optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
optimizer = optim.Adam([var1, var2], lr=0.0001)
```

## Taking an optimization step
모든 옵티마이저는 파라미터를 업데이트하는 step() 메서드를 구현합니다. 이는 두 가지 방식으로 사용할 수 있습니다

### optimizer.step()
이것은 대부분의 옵티마이저에서 지원되는 간소화된 버전입니다. 예를 들어, backward()를 사용하여 그래디언트가 계산된 후에 이 함수를 호출할 수 있습니다.
```python
for input, target in dataset:
    optimizer.zero_grad()
    output = model(input)
    loss = loss_fn(output, target)
    loss.backward()
    optimizer.step()
```

### optimizer.step(closure)
Conjugate Gradient와 LBFGS는 함수를 여러 번 재평가해야 하므로 모델을 다시 계산할 수 있는 클로저를 전달해야 합니다. 이 클로저는 기울기를 초기화하고, 손실을 계산하고, 이를 반환해야 합니다.
```python
for input, target in dataset:
    def closure():
        optimizer.zero_grad()
        output = model(input)
        loss = loss_fn(output, target)
        loss.backward()
        return loss
    optimizer.step(closure)
```

## Base class
> CLASS torch.optim.Optimizer(params, defaults)

### Parameter
- params (iterable): 최적화해야 할 torch.Tensor나 dict의 iterable입니다. 어떤 텐서들을 최적화할 지를 지정합니다.

- defaults (Dict[str, Any]): 최적화 옵션의 기본값을 포함하는 사전입니다. 이는 파라미터 그룹이 특정 옵션을 명시하지 않았을 때 사용됩니다.

### Method
- Optimizer.add_param_group: Optimizer의 param_groups에 파라미터 그룹을 추가합니다.

- Optimizer.load_state_dict: 옵티마이저의 상태를 로드합니다.

- Optimizer.state_dict: 옵티마이저의 상태를 사전 형태로 반환합니다.

- Optimizer.step: 단일 최적화 스텝을 수행하여 파라미터를 업데이트합니다.

- Optimizer.zero_grad: 모든 최적화된 텐서들의 그래디언트를 초기화합니다.
