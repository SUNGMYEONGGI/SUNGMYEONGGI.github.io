# 리스트, 튜플, 딕셔너리, 셋의 각각의 특징

## 리스트(List)
### 장점
- 순서가 있어서 인덱스로 접근 가능
- 중복된 데이터를 저장할 수 있음
- 데이터를 추가, 삭제, 수정이 가능
- 데이터를 슬라이싱, 인덱싱, 정렬이 가능

### 단점
- 데이터 검색 시간이 오래 걸림
- 데이터의 개수가 많아지면 처리 속도가 느려짐

## 튜플(Tuple)
### 장점
- 리스트보다 처리 속도가 빠름
- 데이터를 추가, 삭제, 수정이 불가능
- 데이터를 슬라이싱, 인덱싱, 정렬이 가능

### 단점
- 데이터 검색 시간이 오래 걸림
- 데이터의 개수가 많아지면 처리 속도가 느려짐

## 딕셔너리(Dictionary)
### 장점
- 키와 값으로 데이터를 저장
- 키를 통해 값을 빠르게 찾을 수 있음
- 데이터를 추가, 삭제, 수정이 가능
- 데이터를 슬라이싱, 인덱싱, 정렬이 가능

### 단점
- 중복된 키를 사용할 수 없음
- 데이터의 개수가 많아지면 처리 속도가 느려짐

## 셋(Set)
### 장점
- 중복된 데이터를 저장할 수 없음
- 데이터를 추가, 삭제, 수정이 가능
- 데이터를 슬라이싱, 인덱싱, 정렬이 불가능
